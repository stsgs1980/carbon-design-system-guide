# Справочник дизайн-систем

> Полное руководство по дизайн-системам в стиле **Carbon Design System** от IBM.

**Дизайн-система:** Приложение разработано с использованием принципов Carbon Design System — открытой дизайн-системы IBM для цифровых продуктов.

### Ключевые особенности Carbon

- **IBM Plex Sans** — фирменный шрифт IBM
- **Минимальный радиус** — острые углы (border-radius: 2px)
- **Синий акцент** — IBM Blue (#0f62fe)
- **70ms переходы** — быстрые и точные анимации
- **Модульная сетка** — система отступов кратная 8px

## Обзор

Популярные дизайн-системы и их характеристики.

**Описание:** Одна из самых влиятельных дизайн-систем, созданная для унификации продуктов Google. Включает комплексный набор компонентов, руководств и ресурсов для создания согласованных пользовательских интерфейсов.

- Руководства по анимации
- Фокус на доступность

### Apple Human Interface Guidelines

**Описание:** Фундаментальные принципы и правила разработки в экосистеме Apple. Фокусируется на создании интуитивных, элегантных интерфейсов, которые ощущаются нативными для платформ Apple.

## Принципы

Основные принципы дизайна и методологии.

### Атомарный дизайн

**Описание:** Методология, предполагающая создание систем из атомов (базовые теги), молекул (составные элементы) и организмов (крупные блоки). Разработана Брэдом Фростом для создания масштабируемых и согласованных интерфейсов.

**Уровни:**
- Атомы
- Молекулы
- Организмы
- Шаблоны
- Страницы

### Доступность (Accessibility)

**Описание:** Принцип, заложенный в основу многих систем, таких как Nord, чтобы продуктами могли пользоваться люди с разными возможностями. Включает соответствие стандартам WCAG и инклюзивный дизайн для всех пользователей.

**Аспекты:**
- Соответствие WCAG
- Поддержка скринридеров
- Контраст цветов
- Навигация с клавиатуры

### Дизайн-токены

**Описание:** Визуальные атомы дизайн-системы — именованные сущности, хранящие атрибуты визуального дизайна. Обеспечивают согласованность на всех платформах и упрощают обновление стилей.

**Типы токенов:**
- Цвета
- Типографика
- Отступы
- Тени и радиусы

### Согласованность стиля

**Описание:** Обеспечение единства всех компонентов и визуального языка бренда. Создаёт предсказуемый пользовательский опыт на разных продуктах и платформах, укрепляя узнаваемость бренда.

**Принципы:**
- Единый стиль
- Предсказуемый UX
- Когерентность бренда
- Повторно используемые паттерны

### Масштабируемость

**Описание:** Дизайн-системы должны расти вместе с организацией. Построение с учётом масштабируемости позволяет расширяться и адаптироваться в будущем без потери качества.

**Факторы:**
- Модульная архитектура
- Контроль версий
- Качественная документация
- Управление изменениями

### Документация

**Описание:** Качественная документация — ключ к успешной дизайн-системе. Она должна быть понятной, актуальной и доступной для всех участников команды.

**Компоненты:**
- Руководства по использованию
- Примеры кода
- Лучшие практики
- Часто задаваемые вопросы

## Компоненты

UI-компоненты, организованные по уровню сложности.

### Атомарные компоненты

**Описание:** Наименьшие неделимые элементы UI, служащие строительными блоками для более сложных компонентов. Должны быть максимально гибкими и повторно используемыми.

**Примеры:**
- Кнопки
- Поля ввода
- Чекбоксы
- Радиокнопки
- Иконки
- Лейблы
- Бейджи
- Переключатели

### Молекулярные компоненты

**Описание:** Комбинации атомов, работающих вместе как единое целое. Имеют определённую функцию и могут использоваться самостоятельно в разных контекстах.

**Примеры:**
- Поля форм с лейблами
- Поисковые строки
- Выпадающие меню
- Заголовки карточек
- Группы ввода
- Breadcrumbs

### Организменные компоненты

**Описание:** Сложные секции UI, состоящие из молекул и атомов. Формируют целостные разделы интерфейса с определённой бизнес-логикой.

**Примеры:**
- Панели навигации
- Формы
- Карточки товаров
- Таблицы данных
- Модальные окна
- Сайдбары
- Футеры

### Шаблоны страниц

**Описание:** Структуры на уровне страницы, демонстрирующие расположение компонентов. Показывают реальный контекст использования компонентов.

**Примеры:**
- Макеты дашбордов
- Страницы форм
- Посадочные страницы
- Страницы деталей
- Списки и каталоги
- Статьи

### Визуализация данных

**Описание:** Компоненты для отображения сложных наборов данных в понятном визуальном формате. Критически важны для аналитических интерфейсов.

**Типы:**
- Линейные графики
- Круговые диаграммы
- Карты
- Таймлайны
- Таблицы с сортировкой
- Индикаторы прогресса
- Спарклайны

### Навигационные компоненты

**Описание:** Компоненты для перемещения пользователей по приложению. Включают глобальную и локальную навигацию.

**Примеры:**
- Главное меню
- Табы
- Пагинация
- Хлебные крошки
- Меню пользователя
- Мобильное меню

## Цветовая палитра

Цветовые системы и организация палитры.

### Brand Estonia

**Описание:** Образцовая система цветовой палитры с чёткими руководствами по использованию и учётом доступности. Показывает, как национальная идентичность может быть переведена в цифровой дизайн.

**Структура:**
- Основные цвета бренда
- Вторичные цвета
- Акцентные цвета
- Градиенты
- Цвета для разных медиа

### Цветовые шкалы

**Описание:** Систематические цветовые шкалы от светлого к тёмному для каждого оттенка. Обеспечивают достаточный контраст и гибкость в использовании.

**Особенности:**
- Шкалы 50-950
- Альфа-варианты
- Варианты тёмной темы
- Семантические цвета
- Пользовательские шкалы

### Семантические цвета

**Описание:** Цвета с определёнными значениями для состояний интерфейса и обратной связи пользователю. Должны быть интуитивно понятными.

**Типы:**
- Успех (зелёный)
- Предупреждение (жёлтый/оранжевый)
- Ошибка (красный)
- Информация (синий)
- Нейтральные состояния

### Контраст для доступности

**Описание:** Цветовые комбинации, соответствующие требованиям контраста WCAG. Критически важны для пользователей с нарушениями зрения.

**Требования:**
- Соответствие AA (4.5:1)
- Соответствие AAA (7:1)
- Крупный текст (3:1)
- UI-компоненты
- Фокус-индикаторы

### Тёмная тема

**Описание:** Цветовые адаптации для интерфейсов с тёмной темой. Не просто инверсия цветов, а специально разработанная палитра для комфортного использования.

**Аспекты:**
- Цвета поверхностей
- Система высот (elevation)
- Цвета текста
- Цвета границ
- Акцентные цвета для тёмной темы

## Типографика

Типографические шкалы, шрифты и текстовые стили.

### Шкалы типов

**Описание:** Гармоничная прогрессия размеров шрифтов для согласованной визуальной иерархии. Основана на математических соотношениях для эстетичного результата.

**Уровни:**
- Display (выставочный)
- Заголовки H1-H6
- Основной текст
- Подписи
- Надстрочные элементы
- Мелкий текст

### Шрифтовые семейства

**Описание:** Кураторские шрифты для выражения бренда и обеспечения читаемости. Выбор шрифтов влияет на восприятие бренда и удобство чтения.

**Типы:**
- Основной шрифт
- Вторичный шрифт
- Моноширинный шрифт
- Иконочные шрифты
- Вариабельные шрифты

### Межстрочный интервал и отступы

**Описание:** Вертикальный ритм и отступы для комфортного чтения контента. Влияет на воспринимаемую плотность информации.

**Параметры:**
- Высота строки (line-height)
- Межабзацные отступы
- Межбуквенные интервалы (letter-spacing)
- Оптимальная длина строки
- Висячие строки

### Насыщенность шрифтов

**Описание:** Варианты веса для акцента и создания визуальной иерархии. Слишком много весов могут перегрузить дизайн.

**Значения:**
- Light (300)
- Regular (400)
- Medium (500)
- Semibold (600)
- Bold (700)
- Black (900)

### Адаптивная типографика

**Описание:** Типографика, адаптирующаяся к разным размерам экрана и устройствам. Обеспечивает читаемость на всех платформах.

**Техники:**
- Флюидная типографика
- Viewport-единицы (vw/vh)
- Шкалы для брейкпоинтов
- Минимальные/максимальные размеры
- Контейнерные запросы

## Анимации

Дизайн движения, переходы и микро-взаимодействия.

### Audi Design System

**Описание:** Отличные руководства по движению с наглядными примерами переходов, тайминга и кривых анимации. Демонстрирует точность премиального автомобильного бренда в цифровом движении.

**Аспекты:**
- Функции тайминга
- Стандарты длительности
- Кривые анимации (easing)
- Принципы движения
- Паттерны переходов

### Переходы

**Описание:** Плавные изменения состояния между элементами интерфейса. Делают интерфейс живым и отзывчивым, помогают понять изменения.

**Типы:**
- Затухание (fade)
- Скольжение (slide)
- Масштабирование (scale)
- Раскрытие/сворачивание
- Морфинг
- Комбинированные переходы

### Микро-взаимодействия

**Описание:** Маленькие анимации, обеспечивающие мгновенную обратную связь и создающие ощущение качества. Детали, которые делают продукт особенным.

**Примеры:**
- Hover-эффекты кнопок
- Загрузочные спиннеры
- Анимация переключателей
- Галочки успеха
- Валидация форм
- Курсор и фокус

### Производительность анимаций

**Описание:** Лучшие практики анимации для плавной работы с частотой 60 кадров в секунду. Плохая производительность анимаций убивает пользовательский опыт.

**Рекомендации:**
- GPU-ускорение
- Свойство will-change
- CSS против JS анимаций
- Поддержка reduced-motion
- Оптимизация перерисовок

## Адаптивность

Паттерны адаптивного дизайна и брейкпоинты.

### Система брейкпоинтов

**Описание:** Определённые размеры экрана для адаптивной настройки макетов. Должны соответствовать реальным устройствам пользователей.

**Значения:**
- Мобильные (<640px)
- Планшеты (640-1024px)
- Десктоп (1024px+)
- Широкие экраны (1440px+)
- Телевизоры (1920px+)

### Флюидные макеты

**Описание:** Макеты, которые плавно адаптируются к любому размеру экрана между брейкпоинтами. Уменьшают необходимость в многочисленных контрольных точках.

**Техники:**
- Процентные ширины
- Flexbox
- CSS Grid
- Container queries
- Clamp-функции
- Относительные единицы

### Mobile-First подход

**Описание:** Философия дизайна, начинающаяся с мобильных устройств и масштабирующаяся вверх. Фокусирует на самом важном контенте и функциях.

**Принципы:**
- Прогрессивное улучшение
- Достаточные тач-цели
- Упрощённая навигация
- Фокус на производительности
- Приоритет контента

### Контейнерные запросы

**Описание:** Современная CSS-технология, позволяющая компонентам адаптироваться к размеру своего контейнера, а не viewport. Революция для компонентов.

**Особенности:**
- С-запросы
- Единицы cq
- Адаптивные карточки
- Компонентный подход
- Поддержка браузеров

- Боковая навигация
- Хлебные крошки
